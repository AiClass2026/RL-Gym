# 连续动作空间 PPO：贪心评估 vs 采样评估

## 背景

在 CarRacing-v3 上使用 PPO（连续动作空间，高斯策略）训练后，发现：

- **采样评估**（从高斯分布中采样动作）：eval_reward 可达 840+
- **贪心评估**（直接使用策略均值作为动作）：eval_reward 大部分时间在 -71 左右

贪心策略的表现远不如采样策略，这看起来违反直觉——均值不就是"最可能"的动作吗？

## 核心原因：PPO 优化的是分布的期望回报，不是均值动作的回报

PPO 的策略梯度目标是：

$$\max_{\mu, \sigma} \; \mathbb{E}_{a \sim N(\mu, \sigma)} \left[ A(s, a) \right]$$

μ 和 σ 是**一起优化的**。最终学到的 μ 不是"最优动作"，而是"在给定 σ 下使分布期望回报最大的那个中心点"。这两件事不等价。

## 三个具体因素

### 1. 动作裁剪（clamp）使实际分布偏离高斯均值

代码中采样后会裁剪到合法范围：

```python
action_raw = dist.sample()                                        # 可能超出范围
action_clipped = torch.clamp(action_raw, action_min, action_max)  # 裁到 [min, max]
```

以油门（范围 [0, 1]）为例，假设 μ=0.3, σ=0.3：

- 原始高斯均值 = 0.3
- 但裁剪后，所有 <0 的样本被推到 0，>1 的被推到 1
- **裁剪后分布的实际均值 ≈ 0.38**（高于高斯均值）

训练时策略梯度针对**裁剪后的动作**计算，策略会利用这个偏移——可能故意把 μ 设偏低，"知道"裁剪会把实际执行的均值抬上去。贪心直接用 μ=0.3，就比采样+裁剪的效果差了。

### 2. 标准差较大时，均值只是分布的"模糊中心"

从训练数据可以计算最终的策略标准差：

```
loss_entropy ≈ 0.0048
→ entropy = 0.0048 / 0.01 = 0.48
→ 3 个动作维度的平均 σ ≈ 0.28
```

σ=0.28 相对于动作范围（1~2）来说相当大，策略还远未收敛到确定性策略。均值只是一个"模糊中心"，采样出来的多样化动作才是策略真正依赖的行为。

### 3. 噪声在轨迹层面提供误差修正

单步来看，采样有约一半概率比均值更差。但推广到 1000 步的完整 episode：

- **贪心**：如果均值动作的转向系统性偏小（哪怕只差 5%），误差在每步累积，50 步后车对着草地开。而且一旦偏离赛道，均值动作没有任何机制纠正。
- **采样**：噪声是每步独立的。连续 3 步采到偏小转向后，第 4 步可能采到偏大转向，形成自然的误差修正。轨迹在赛道上蛇形抖动，但不会系统性偏离。

类比走钢丝：一个人步伐固定但重心偏左 1cm，走 50 步后掉下去；另一个人每步随机偏左或偏右 5cm，看起来摇摇晃晃，但因为平均居中，反而能一直走下去。

## 不对称奖励加剧了均值偏移

弯道场景下，奖励函数往往不对称：

- 油门过大（>0.9）→ 打滑冲出赛道，**惩罚 -50**
- 油门过小（<0.2）→ 车太慢，惩罚 -3

在这种不对称惩罚下，策略梯度会把 μ 往**保守方向**推：

| μ | σ | 灾难概率 (>0.9) | 期望回报 |
|---|---|---|---|
| 0.6（最优动作）| 0.3 | ~16% | ≈ -1.7 |
| 0.4（保守偏移）| 0.3 | ~5% | ≈ +2.5 |

PPO 收敛到 μ=0.4 而非 0.6。贪心执行 μ=0.4 就比实际最优偏保守了。

## 与离散动作空间的对比

| | 连续（高斯） | 离散（Categorical） |
|---|---|---|
| 策略参数 | μ, σ（间接控制动作） | 每个动作的概率 p(a) |
| 优化方式 | 调整分布形状间接影响采样 | **直接**提高好动作的概率 |
| 贪心含义 | 取高斯均值（分布中心） | 取概率最高的动作 |
| 贪心效果 | 可能偏离最优 | 通常接近最优 |

离散动作空间中，策略为每个动作**单独**分配概率，好动作的概率会被直接推高，argmax 拿到的就是最好的。连续空间只能控制一个钟形曲线的位置和宽度，无法对某个具体值"单独加分"，所以曲线顶点（均值）不一定是最优动作点。

## 实践建议

### 当前 PPO + 连续高斯策略

评估时应使用**采样模式**（`greedy=False`），因为训练就是在采样模式下进行的，采样评估才能真实反映策略能力。

### 想让贪心评估也有效？

有以下几个方向：

1. **训练更久，让 σ 自然下降**：当 σ 足够小（如 <0.05）时，采样 ≈ 均值，greedy 自然有效。
2. **减小熵系数**：降低 `entropy_scale`，减少对探索的鼓励，加速 σ 收敛。
3. **换用确定性策略算法**：DDPG、TD3、SAC 直接优化确定性策略 μ(s)，从设计上就不存在 μ-σ 耦合问题，greedy 天然就是最优的。

### 改用离散动作？

CarRacing-v3 支持 `continuous=False`（5 个离散动作），greedy 评估会正常工作，但因为控制精度不足（不能组合动作、没有中间值），最终成绩通常只有连续版的一半左右（200~400 分 vs 800+ 分）。
